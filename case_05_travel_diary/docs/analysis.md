# Анализ выполненной кейс-задачи

1. Реализовано web-приложение "Дневник путешествий" с backend на FastAPI и интерфейсом на HTML/CSS.
2. Добавлена регистрация пользователя и авторизация через cookie-сессию.
3. Реализована функция создания записи о путешествии с названием, описанием и признаком публичности.
4. Добавлен публичный просмотр путешествий других пользователей на главной странице.
5. Реализована геопривязка путешествия через поля широты и долготы.
6. Добавлены изображения мест через URL, стоимость путешествия, места культурного наследия и места для посещения.
7. Реализованы оценки путешествия по удобству передвижения, безопасности, населенности и природной составляющей.

## Рекомендации по устранению и профилактике ошибок

1. Перед запуском проверять доступность портов `8005` и `5435`.
2. Для демонстрации использовать seed-данные, чтобы сразу проверить публичную ленту и разные аккаунты.
3. Добавить загрузку изображений на сервер или в объектное хранилище вместо хранения только внешних URL.
4. Добавить проверку диапазонов широты и долготы на сервере.
5. Добавить редактирование и удаление записей путешествий пользователем.
6. Реализовать фильтрацию путешествий по городу, стоимости и оценкам.
7. Расширить тестовое покрытие для авторизации, создания записей, публичности и валидации оценок.
