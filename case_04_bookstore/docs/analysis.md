# Анализ выполненной кейс-задачи

1. Реализована web-версия книжного магазина с серверной частью на FastAPI и клиентским интерфейсом на HTML/CSS.
2. Добавлена ролевая модель пользователей: обычный пользователь работает с каталогом, администратор управляет книгами.
3. Создан каталог книг с хранением данных в PostgreSQL: название, автор, год, категория, описание, цена и статус доступности.
4. Реализована сортировка каталога по названию, автору и году написания, а также фильтрация по категории.
5. Добавлена покупка книги пользователем с фиксацией операции в таблице покупок и изменением статуса книги.
6. Реализована аренда книги на 2 недели, 1 месяц или 3 месяца с расчетом даты окончания аренды.
7. Добавлены административные функции: создание книги, изменение цены, изменение статуса доступности и просмотр напоминаний об окончании аренды.

## Рекомендации по устранению и профилактике ошибок

1. Перед запуском проверять доступность портов `8004` и `5434`.
2. Для демонстрации проекта использовать seed-данные, чтобы сразу проверить роли администратора и пользователя.
3. При развитии проекта заменить автоматическое создание таблиц на миграции Alembic.
4. Добавить серверную валидацию диапазона года написания и минимальной цены книги.
5. Реализовать возврат книги пользователем, чтобы статус книги автоматически снова становился `available`.
6. Расширить механизм напоминаний: добавить отправку email или отдельный журнал отправленных уведомлений.
7. Добавить тесты для покупки, аренды, смены статусов, прав администратора и фильтрации каталога.
